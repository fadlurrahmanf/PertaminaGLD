Key Observations

MQ135 — Primary CO2 Indicator
Dominant for CO2 (0.313V, 79.5 dB SNR). Also strong for LPG at high concentration. 6x stronger than next sensor for CO2.
MQ2 — Primary LPG Indicator
Dominant at low LPG concentration. Saturates at high concentration. Also responds to O2.
MQ8 / MQ3 — Blind Sensors
Near-zero response to all gases. Useful as negative controls — if they respond, it's not the target gas.

ML Architecture Recommendations

Recommended: TCN (Temporal Convolutional Network)

Why TCN over LSTM
- Lag-1 autocorrelation: 0.992+ (excellent temporal memory)
- Low entropy (0.02-0.46) = highly predictable signals
- 1 Hz sampling oversamples slow dynamics (T90 = 38-180s)
- Faster training and inference than RNNs
- INT8 quantized for ESP32 deployment
Architecture
- Input: 8 raw voltages + 6 engineered = 14 features
- Window: 30-60 samples (30-60 seconds)
- Output: 7 classes (CLEAR, LPG, METHANE, PROPANE, BUTANE, O2, CO2)
- Quantization: INT8 for ESP32 inference

Conclusions

1. GLD is the reference system. ADS1256 24-bit ADC reveals true MQ sensor differentiation that Chamber's 12-bit ADC cannot see. 1000:1 vs 3.5% sensor spread.
2. Each gas has a unique fingerprint. LPG: MQ2/MQ135 dominant, concentration-dependent. CO2: MQ135 dominant, all sensors correlated. O2: MQ6 dominant, weak signal.
3. Rise time is the best discriminator. LPG (8-222s), CO2 (38-3307s), O2 (38-180s). Rate-of-change (dV/dt) separates gas types better than raw voltage.
4. TCN is the recommended architecture. Smooth signals, excellent temporal structure (autocorrelation 0.992+), 14-feature input, 7-class output, INT8 quantized for ESP32.
5. Data collection is the bottleneck. Need 20+ runs per gas class (140+ total). Currently have 7 runs across 3 gases. Clean air baseline is the most critical missing class.
6. Port Chamber ML tools to GLD. Auto-labeling, 5-method validation, model export — all valuable, all need GLD data to produce meaningful results.
