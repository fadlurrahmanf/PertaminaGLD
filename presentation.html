  • Kebocoran gas LPG/beracun di lingkungan industri & rumah tangga
  • Deteksi dini menyelamatkan jiwa & aset
  • Area luas membutuhkan banyak sensor tersebar

  • Server Node-RED untuk monitoring terpusat

Topologi Jaringan

LoRa STAR

  • GLD Nodes → Cluster Head
  • Point-to-multipoint
  • GLD tidak perlu routing
  • CH sebagai penerima tunggal

LoRa MESH

  • Cluster Heads ↔ Gateway
  • Multi-hop routing
  • Coverage area luas
  • Fallback jalur alternatif
F001 \ / CH1 —- CH2
F002 -- STAR -- CH3 <--MESH--> |
F003 / \ CH4 —- CH5 — GW —TCP/MQTT— Server

Alur Data End-to-End

  1. GLD Node: Baca 8 sensor → ML inference → enkripsi AES-128-GCM → kirim via LoRa STAR
  2. Cluster Head: Terima → cache record → siapkan relay MESH
  3. CH → Gateway (MESH): Forward data via LoRa MESH multi-hop
  4. Gateway: Terima MESH → parse frame → publish MQTT
  5. Server Node-RED: Subscribe MQTT → dekripsi → dedup → simpan MySQL/CSV
  6. Alarm Push: GLD deteksi gas → alarm flag → CH relay prioritas → server notifikasi
Sensor → ADC → ML → Encrypt → LoRa → CH → MESH → GW → MQTT → Node-RED → Decrypt → DB

Keamanan Data

AES-128-GCM

  • Enkripsi payload sensor di GLD Node
  • Dekripsi di server Node-RED
  • Key 128-bit, IV 12-byte random
  • Authentication tag (integrity check)
  • Tidak ada decrypt di CH atau Gateway

Format Payload

GCM_tag(16) + IV(12) + ciphertext

Frame header: srcId(2) + dstId(2) + seq(2) + flags(1) +...
