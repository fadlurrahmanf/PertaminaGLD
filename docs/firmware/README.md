## Purpose

- `01-gld-reference-traceability.draft.md`: membandingkan kontrak baru dengan firmware lama dan menentukan apa yang dipakai, diadaptasi, ditolak, atau ditahan.
- `02-gld-3-stage-plan.draft.md`: mengunci behavior GLD untuk running/inference, dataset/training capture, dan nulling/setup calibration.
- `03-phase-1-plan.draft.md`: rencana implementasi GLD-CH uplink MVP berdasarkan kontrak payload.
- `04-phase-1-implementation-checklist.draft.md`: checklist eksekusi coding phase 1 setelah user memberi approval eksplisit.
- `05-firmware-versioning-backup-policy.draft.md`: aturan versi firmware, build metadata, changelog, dan backup hemat storage untuk rollback.
